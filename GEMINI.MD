# Product Name Normalizer - Project Context & Rules

## Project Overview
This project is an industrial-grade tool for normalizing tens of thousands of product names using LLMs (specifically Spark/OpenAI-compatible APIs).

## Core Architecture & Patterns
- **Async Concurrency**: Controlled via `asyncio.Semaphore(CONCURRENCY_LIMIT)`.
- **Immediate Checkpointing**: Every successful result is appended to `result_checkpoint.jsonl` **immediately** after task completion. This is a non-negotiable safety feature to prevent data loss.
- **Resumption Logic**: On startup, the script must load `result_checkpoint.jsonl` to deduplicate and skip already processed items.
- **Logging Split**: 
  - `process.log`: Operational info.
  - `model_request.log`: Raw API request/response dump (for auditing).

## Development Guidelines
- **Libraries**: Use `loguru` for logging, `pandas` for Excel/data manipulation, and `tqdm.asyncio` for progress visualization.
- **Error Handling**: Implement multi-turn retry logic for API calls. If an item fails all retries, log it and save to `failed_products.txt` for next-round processing.
- **JSON Robustness**: Use regex to extract JSON blocks from model responses to handle cases where the model includes extra text or markdown markers.
- **Data Integrity**: Never overwrite the source `product.xlsx`. The final output should always be `result.xlsx`, generated by aggregating `result_checkpoint.jsonl`.

## Operational Commands
- Run: `python main.py`
- Monitor: `tail -f process.log`
- Background: `nohup python main.py > nohup.out 2>&1 &`

## Environment Dependencies
Required keys in `.env`:
- `SPARK_API_KEY`
- `SPARK_BASE_URL`
- `SPARK_MODEL_DOMAIN`
